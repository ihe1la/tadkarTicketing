Security Assessment — Executive Summary

Tadkar Ticketing Platform  ·  wirafpay.com
RISK: CRITICALImmediate action required
White-box review · 29 Jun 2026 · Confidential

The bottom line

The platform was built as a local demo prototype with authentication intentionally left out — an acceptable choice for a developer's machine. The critical problem is that this prototype is now published on the public internet. In that setting, the missing authentication means any anonymous visitor can read all customer data, impersonate any employee or manager, and alter any record. No password, vulnerability, or special skill is required.

Key findings

SevIssue
CRITNo authentication. Identity is just a value sent by the browser; the server trusts it blindly.
CRITAccount list is public. One open URL returns every user and the "key" needed to impersonate them.
CRITFull data dump. A single request returns every ticket, message, and customer's personal data.
HIGHWeak access rules. Several roles can view records they should not — even if logins were added.
HIGHForgeable history. Actions can be logged under any employee's name; the audit trail is untrustworthy.
HIGHPrivilege spoofing. A visitor can simply claim the "manager" role to unlock all reports and actions.

Business impact

  • Data breach: exposure of all customer PII and private support conversations.
  • Privacy / regulatory: unauthorized disclosure and a logging trail that cannot be relied on for disputes.
  • Integrity: tickets can be closed, edited, or fabricated by outsiders.
  • Reputation: trivially exploitable; discoverable by routine internet scanning.

Reassurance — what is solid

No SQL injection, no cross-site scripting, no leaked passwords or secret keys, and no insecure third-party data handling were found. The exposure stems from one root cause — missing authentication — not widespread poor coding.

Immediate action

Take the prototype off the open internet today — or place it behind a VPN, IP allow-list, or simple login gate. This single step neutralizes all three critical issues at once and buys time to add proper authentication safely.
